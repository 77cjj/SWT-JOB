# Docs 语义重整执行包

本文档用于将 `src/pages/docs/**/*.mdx` 全量做语义审查、去重整合、命名统一与结构重写。目标风格对齐 `src/pages/docs/after/shopping.mdx`：信息密度高、层级清晰、可执行。

## 1) 语义审查规则（Audit Rules）

### 标题层级与信息价值
- 每篇文档必须且仅有一个 `# H1`，用于表达“该文档唯一主题”。
- `## H2` 只保留“高价值信息分组”，例如：适用场景、步骤、材料、注意事项、风险、FAQ。
- `### H3` 仅用于 H2 下需要拆分时使用，不允许层级跳跃（禁止 H1 后直接 H3）。
- 删除无信息增量的标题（如“今天给大家奉上”“看了这篇就…”）。

### 噪声与口语化清理
- 删除以下噪声词/句式：`左滑查看`、`图片左滑查看`、`超详细教程`、`超详细攻略`、情绪化口号、重复感叹。
- 删除营销化/主播口吻（如“保姆级”“分分钟学会”“不看后悔”），改为中性指导语。
- 删除排版噪声（如 `## ##`、连续空标题、无正文标题）。

### 内容质量与一致性
- 优先保留“可执行信息”：条件、步骤、时间点、材料、费用、限制、风险、例外。
- 删减泛泛描述和重复背景，避免同义反复。
- 数字、单位、货币、时态统一（如 `$`, `%`, `kg`, `Wh`）。
- 英文产品名保留官方写法（如 `Google Meet`, `Skype`, `Zoom`, `Microsoft Teams`）。

### 术语规范
- 统一术语：`行李清单`、`签证材料`、`面签`、`入境`、`保险`、`社安号 (SSN)`、`Job Offer`、`DS-2019`、`SEVIS`。
- 同义词归并：例如“行李准备清单/赴美行李准备清单/打包清单”归一为“行李清单”。
- 用词避免绝对化（如“必须”“绝对”）除非确属法规要求。

## 2) 重复与近似内容整合策略（Merge Policy）

### A. 完全重复（重复率 >= 85%）
- 合并为一篇主文档，保留更完整、更新、结构更清晰的版本。
- 另一篇内容并入主文档后删除冗余段落，必要时保留“补充说明”小节承接差异点。
- 示例：`preparation/packing.mdx` 与 `preparation/pre-departure/packing/doc-017.mdx` 合并为一篇标准化“行李清单”。

### B. 主题近似（重复率 40% - 85%）
- 放在同一级导航下，不强制硬合并成一篇。
- 若多个文档是同一类工具/平台，做“并列分组”：
  - 例如 `Google Meet / Skype / Zoom / Teams` 归为同一父级主题（线上沟通工具）。
- 统一写法为“总览 + 子项对比 + 操作步骤 + 常见问题”。

### C. 差异信息保护
- 任何文档中的特有限制条件、地区差异、时间窗口、材料要求不得丢失。
- 若合并后存在冲突信息，写成“条件分支”或“版本说明”，不要静默覆盖。

### D. 导航与命名规则
- 文件名、slug、标题保持一一对应，避免“标题讲A、slug像B”。
- 命名短而准：优先“主题词 + 场景词”（如“行李清单”“线上面试工具”“美国公交”）。
- 去除夸张修饰词和无意义前后缀。

## 3) 标准文档模板规范（Format Spec）

### 通用结构
```mdx
# 文档标题（主题唯一、名词化）

一句话摘要：谁在什么场景下为什么要看这篇。

## 适用场景
- 人群：
- 阶段：
- 目标：

## 核心内容（按主题拆分为 2-5 个 H2）
### 小节（可选）
- 分点说明

## 操作步骤 / 执行清单
1. 步骤一
2. 步骤二
3. 步骤三

## 材料清单（满足条件时用表格）
| 材料 | 是否必需 | 用途 | 获取方式 | 备注 |
|---|---|---|---|---|
| 护照 | 必需 | 身份证明 | 自备 | 需在有效期内 |

## 注意事项与风险
- ...

## 常见问题（可选）
### Q1
A1
```

### 表格触发规则
- 当内容属于“材料清单 / 参数对比 / 平台对比 / 时间线节点”时，优先表格。
- 当内容属于“解释、步骤、建议”时，优先分点或编号列表。
- 同一章节中避免无意义大表，控制在“可扫描、可执行”的粒度。

### 禁止写法
- 禁止“左滑查看”“图片左滑查看”“超详细教程”等噪声文本。
- 禁止连续空标题、重复标题、标题即废话。
- 禁止纯情绪化段落占据正文。
- 禁止大段无结构长文（超过 6 行建议拆为分点）。

## 4) 主 Prompt（可直接投喂模型执行全量重写）

```text
你是一个“文档语义审查与重构编辑器”。请对项目中的 docs 文档执行全量语义整理与标准化重写。

【输入范围】
- 处理范围：src/pages/docs/**/*.mdx
- 参考风格：src/pages/docs/after/shopping.mdx
- 导航/消费约束参考：src/lib/docs/legacy.ts, src/lib/docs/types.ts

【核心目标】
1) 每篇文档只保留有信息增量的一二级标题（H1/H2），层级清晰。
2) 语义与格式标准化：去噪、去口语化、去营销化、去重复。
3) 合并重复文档（尤其“行李准备清单”类），近似主题归入同一级并列结构。
4) 对“材料要求/参数对比/时间节点”使用表格呈现，其余用分点和步骤表达。
5) 输出内容务必可执行、可扫描、可维护。

【硬性规则】
- 每篇文档必须且仅有一个 H1。
- H2 必须是“有用信息模块”，禁止空洞标题。
- 删除所有“左滑查看/图片左滑查看/超详细教程/看完就会”等噪声句式。
- 修复异常标题格式（例如“## ## ...”）。
- 统一术语：行李清单、签证材料、面签、入境、保险、社安号(SSN)、Job Offer、DS-2019、SEVIS。
- 英文工具名统一官方写法：Google Meet, Skype, Zoom, Microsoft Teams。
- 保留且突出关键事实：条件、步骤、材料、费用、限制、风险、例外。
- 不得丢失法规/流程关键约束；若信息冲突，显式写“适用条件/版本差异”。

【去重与归类策略】
- 重复率 >= 85%：合并为一篇主文档，保留结构更优版本，另一篇差异内容并入“补充说明”。
- 重复率 40%-85%：不强制合并，但必须放在同一级主题下并列组织。
- 工具类文档（如 Google Meet/Skype/Zoom/Teams）统一归入“线上沟通工具”主题簇，采用一致模板。

【统一模板】
- 开头：一句话摘要（人群 + 场景 + 目标）
- 必备模块：
  - 适用场景
  - 核心内容（2-5 个 H2）
  - 操作步骤/执行清单
  - 注意事项与风险
  - 常见问题（可选）
- 当涉及材料要求时，必须增加“材料清单表格”，列包含：
  - 材料 | 是否必需 | 用途 | 获取方式 | 备注

【命名与可读性】
- 文档标题简洁名词化，去除夸张修饰词。
- 文件名/slug/标题语义一致。
- 段落短句化；每段不超过 5 行；优先列表与编号步骤。

【输出要求】
请按“文件级改写结果”输出：
1) 文件路径
2) 重写后标题
3) 是否合并（若合并，写明来源文件）
4) 重写后的完整 MDX 内容
5) 本文件执行的规则清单（命中哪些规范）

最后追加“全局变更报告”：
- 合并了哪些重复文档
- 建立了哪些同级主题簇
- 清理了哪些噪声模式
- 新增了哪些表格化清单
- 仍需人工确认的冲突点
```
