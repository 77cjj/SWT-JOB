## 2. P0 — 发布前必做

### 2.1 数据与持久化

- [ ] **市集钱包 / 帖子 / 个人资料** 当前写在 Vercel `/tmp` 或本地 `.data`，**实例重启即丢**。生产必须迁到 **Postgres（或至少 Redis+DB）**，与后端用户表 `userId` 对齐。
- [ ] Stripe Webhook 幂等入库（`session_id` / `payment_intent` 唯一约束），禁止仅写文件。
- [ ] 备份与回滚：DB 日备、迁移脚本可重复执行（已有 `scripts/db-migrate.sh` 需纳入发布 checklist）。

### 2.2 认证与授权

- [ ] 生产关闭 `NEXT_PUBLIC_RAGENT_BYPASS_AUTH` / 开发后门密码。
- [ ] Google OAuth：确认 **redirect** 模式、回调 URL、生产 Client ID；验证管理员账号可打开 `/u/{id}` 与钱包（不再 Login required）。
- [ ] Sa-Token：`/user/me`、`/user/password`、`/user/chat-credits/redeem` 需登录；公开接口最小化。
- [ ] 管理后台路由服务端鉴权（不仅前端隐藏菜单）。

### 2.3 支付与资金安全

- [ ] Stripe **pk/sk/whsec 仅 Vercel**，密钥轮换流程；Webhook 签名校验已接入需在生产实测充值到账。
- [ ] 钱包扣款与 AI 额度：**先兑后扣或事务化**（本轮 buy-ai 已改为先 redeem 成功再扣余额）。
- [ ] 明确退款 / 争议 / 关闭帖子退押金规则，写入用户协议摘要页。
- [ ] 设置单笔/日累计充值与提现上限（防盗刷）。

### 2.4 核心路径冒烟（上线当天）

1. Google / 账号密码登录 → 头像菜单三项正确。  
2. 打开**自己的**个人主页 → 保存资料与隐私 → 改密。  
3. 钱包充值（Stripe 或演示）→ 购买 AI 次数 → AI 问答扣次。  
4. 市集发帖（余额不足时拒绝）→ 编辑 → 暂停 → 恢复 → 关闭 → 再恢复（需有余额）。  
5. 买家接单 → 凭证 → 确认 → 放款流水正确。  
6. 站内留言推送微信通畅（无「需配置 PushPlus」外露文案）。  
7. 管理后台入口仅管理员可见且可进各业务页。

### 2.5 运维发布

- [ ] 后端：`git pull` + `./server.sh restart backend --build --force`（必要时 `db up`）。  
- [ ] 前端：Vercel Redeploy production；确认 `NEXT_PUBLIC_RAGENT_API_BASE_URL` 指向当前 ECS。  
- [ ] 健康检查：后端 9090、Nginx、磁盘、内存（4GB 机用 `standard` profile）。  
- [ ] 错误监控：至少 Sentry 或等价（前后端）+ 关键日志保留。
